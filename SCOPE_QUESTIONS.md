# SCOPE_QUESTIONS

実装中に発生したスコープ判断の記録。判断に迷う拡張は実装せず、ここに記録して最小解釈で先へ進む。

---

## 1. 未決済時の content_json 生成タイミング（M4）

仕様6は「PDF全文生成の直前に Stripe Checkout」、同時に「プレビューは概要書の1セクションのみ透かし付きで表示」を要求する。プレビューには生成済みの文書内容が必要なため、**content_json（LLM生成）は無料段階で実行し、PDF化のみ決済後に解禁**という最小解釈を採用。LLMコストは1回分先行するが、プレビューが実文書になり営業導線（仕様6の意図）が強くなる。

## 2. E2Eにおける決済ステップ（M5）

StripeのホストされたCheckoutページの自動操作は変更に弱くфлakyなため、E2Eでは**Webhook処理相当の状態遷移（projects.status='paid'）を service role で直接実行**して代替。Webhook本体（署名検証・冪等性・遅延決済）は `lib/stripe/__tests__/webhook.test.ts` の単体テストで担保。テストモードの手動決済確認手順は PROGRESS.md M4 に記載。

## 3. LPの「デモGIF」（M5）

仕様のLP構成は「課題→デモGIF→価格→CTA」。GIF収録は Supabase 接続後でないと実画面が動かないため、当面は**1問1画面UIの静的な再現モック**をLPに置いた。実データ接続後、画面収録GIFに差し替えるのが望ましい。

## 4. インタビュー再開時の未回答深掘り質問

セクション途中で深掘り質問に答えず離脱した場合、再開時は次セクションから始まり、未回答の深掘りは再提示されない（回答済み扱いにもならず、文書生成時は「未回答」として扱われる）。再提示ロジックは仕様外の拡張と判断し実装せず。
